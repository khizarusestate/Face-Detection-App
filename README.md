# 🎯 Face Recognition System

A real-time, browser-based face recognition web application built with MediaPipe, TensorFlow.js, and HTML5 Canvas. Detect, enroll, and recognize faces directly in your browser without any backend server.

## ✨ Features

### Core Functionality
- 📹 **Real-time Face Detection** - Continuous face detection from webcam feed
- 🎓 **Face Enrollment** - Store multiple face samples with names
- 🔍 **Face Recognition** - Real-time face matching against enrolled database
- 🧠 **Feature Extraction** - Generates face embeddings (256-dimensional vectors) from detected landmarks
- 💾 **Local Storage** - All face data stored in browser's localStorage
- ✅ **Similarity Matching** - Euclidean distance-based matching with confidence scores

### User Interface
- 🖼️ Live webcam stream with bounding boxes
- 🎨 Green bounding boxes around detected faces
- 📊 Real-time confidence and statistics display
- 🗂️ Database viewer to see enrolled faces
- 🎛️ Simple, intuitive control panel
- 📱 Responsive design (works on desktop and tablets)

### Tech Stack
- **Face Detection**: MediaPipe Face Detection API
- **Embedding Generation**: Custom feature extraction from face landmarks
- **Similarity Matching**: Euclidean distance and cosine similarity
- **Storage**: Browser localStorage
- **Canvas API**: Real-time drawing and visualization

## 🚀 Getting Started

### Prerequisites
- Modern web browser (Chrome, Firefox, Safari, Edge) with:
  - WebRTC support (for camera access)
  - localStorage support
  - JavaScript enabled

### Installation

1. **Navigate to the app directory**:
```bash
cd face-recognition-app
```

2. **Open in a browser**:
   - Simply open `index.html` in your web browser
   - Or use a local server:
   ```bash
   # Using Python
   python -m http.server 8000
   
   # Using Node.js (http-server)
   npx http-server
   
   # Using Vite (if available in your project)
   npx vite
   ```

3. **Grant Camera Permissions**:
   - When prompted, allow the browser to access your camera
   - Required for live video feed

### First Run

1. **Enroll Faces**:
   - Enter a person's name in the input field
   - Click "Add Face Sample" button
   - Look directly at the camera for 3 seconds while the app captures your face
   - You should see a green border/notification when enrolled
   - Repeat for multiple people (more samples = better recognition)

2. **Start Recognition**:
   - Click "Start Recognition" button
   - The app will continuously compare detected faces against enrolled faces
   - Matches will display above the detected face with confidence percentage
   - "Unknown" appears if no match found

3. **Manage Database**:
   - Click "View Database" to see all enrolled faces
   - Delete individual faces or click "Reset Database" to clear all

## 📚 How It Works

### Face Detection
The app uses **MediaPipe Face Detection** to:
- Detect face presence in real-time video stream
- Extract facial keypoints/landmarks
- Provide bounding box coordinates

### Embedding Generation
Face embeddings are generated by extracting geometric and spatial features:
- Landmark coordinates (normalized positions)
- Pairwise distances between landmarks
- Angles between landmark triplets
- Features are normalized to unit length (L2 normalization)

### Face Matching
When recognizing a face:
1. Detect face in current video frame
2. Generate embedding from detected landmarks
3. Compare with average embeddings of enrolled people
4. Calculate Euclidean distance to each enrolled person
5. Find closest match with confidence score
6. Display name if confidence exceeds threshold (default: 50%)

### Data Storage
- All face embeddings stored in browser's localStorage
- No data sent to server
- Embeddings are compact (256 floats ≈ 1KB per face)
- Data persists between browser sessions

## ⚙️ Configuration

### Similarity Threshold
Location: `script.js` line ~280
```javascript
const match = faceDatabase.findClosestMatch(embedding, threshold = 0.5);
```
- **Lower threshold** (0.3-0.4): More lenient, more false positives
- **Higher threshold** (0.6-0.8): More strict, more false negatives

### Enrollment Timeout
Location: `script.js` line ~237
```javascript
const maxAttempts = 30; // 3 seconds at 10fps
```
- Increase for more time to detect face during enrollment

### Embedding Dimension
Location: `script.js` line ~135
```javascript
this.inputSize = 256; // Output embedding size
```
- Larger size = more distinctive faces but more storage

## 🐛 Troubleshooting

### Camera Not Working
- Check browser permissions (Settings → Camera)
- Ensure no other app is using the camera
- Try a different browser
- Check camera is working in OS settings

### No Faces Detected
- Ensure good lighting
- Face should be clearly visible
- Try getting closer to camera
- Check camera resolution

### Poor Recognition Accuracy
- Enroll faces in different angles
- Add multiple samples per person (3-5)
- Ensure faces are clearly visible
- Check lighting consistency

- Browser has ~5-10MB limit per domain
- Delete old enrollments or use "Reset Database"
- Clear browser data if needed

## 🔄 Algorithm Details

### Feature Extraction
```
Input: Face landmarks (6-8 key points)
  ↓
Extract coordinates, distances, angles
  ↓
Combine into 256-dimensional feature vector
  ↓
Normalize to unit length (L2 norm)
  ↓
Output: Face embedding
```

### Matching
```
Input: Face embedding
  ↓
For each enrolled person:
  Calculate Euclidean distance to their average embedding
  ↓
Find minimum distance (best match)
  ↓
Convert distance to confidence score
  ↓
Compare with threshold
  ↓
Output: Matched name or "Unknown"
```

## 💡 Tips for Best Results

1. **Good Lighting**: Bright, even lighting without harsh shadows
2. **Face Position**: Look directly at camera, face centered in frame
3. **Multiple Samples**: Enroll 3-5 samples per person for better accuracy
4. **Distance**: Stay 30-60cm from camera
5. **Angles**: Enroll faces at different angles (front, slightly left/right)
6. **Consistency**: Recognition works best with similar conditions to enrollment
